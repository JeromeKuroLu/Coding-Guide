JavaScript编码规范

使用ES6中的规范

  • 使用对象方法的简写
  • 使用对象属性的简写
  • 在对象属性声明前把简写的属性分组

因为这样能清楚地看出哪些属性使用了简写

  • 使用拓展运算符 ... 复制数组

文件与编码

  • 静态常量分离，命名使用 “大写”+”_”
  • 字符串使用单引号 ''

解构

  • 使用解构存取和使用多属性对象

解构能减少临时引用属性

  • 对数组使用解构赋值
  • 需要回传多个值时，使用对象解构，而不是数组解构

增加属性或者改变排序不会改变调用时的位置

函数

  • 使用函数声明代替函数表达式

因为函数声明是可命名的，所以他们在调用栈中更容易被识别

此外，函数声明会把整个函数提升（hoisted），而函数表达式只会把函数的引用变量名提升

  • 直接给函数的参数指定默认值，不要使用一个变化的函数参数
  • 当需要传递一个匿名函数，或使用函数表达式时，使用箭头函数

属性与变量

  • 使用 . 来访问对象的属性
  • 一直使用 const 来声明变量

避免全局命名空间的污染

  • 对于不确定值的变量。可以初始化赋值null，通过 value !== null 判断

比较运算符

  • use !== & === instead of != & ==

  • 使用简写

代码美观

  • 使用 // 作为单行注释。在评论对象上面另起一行使用单行注释。在注释前插入空行
  • 注释前缀

对于需要简短说明的注释，在注释前使用前缀快速标注

FIXME 代表需要 review 的代码段/标注问题

TODO 则标注问题的可能解决方案

  • 在花括号前放一个空格
  • 在控制语句（if、while 等）的小括号前放一个空格
  • 在使用长方法链时进行缩进。使用前面的点 . 强调这是方法调用而不是新语句

null & undefined

  • 不要用null来检测是否传入了某个参数
  • 不要用null来检测一个未初始化的变量
  • 把null当做占位符，初始化变量的时候赋值
  • 如果所期望的值真的会是null，则可以直接和null比较
  • 检测 函数/原始值 使用 typeof
  • 检测 引用值/自定义类型 用 instanceof

命名规则

  • 使用小驼峰式命名对象、函数和实例
  • 使用大驼峰式命名构造函数或类
  • 使用下划线 _ 开头命名私有属性
  • 别保存 this 的引用。使用箭头函数
  • 如果你的文件只输出一个类，那你的文件名必须和类名完全保持一致

jQuery相关

  • 使用 $ 作为存储 jQuery 对象的变量名前缀
